# Ember Run

> An atmospheric runner set in darkness. You're alone in the shadows, racing to collect 10 glowing lanterns. Each lantern lights your path, but danger surrounds you. How far will you go?

[![Play Now](https://img.shields.io/badge/Play-Now-brightgreen)](https://vcookie-ember-run.netlify.app)
[![GitHub Repo](https://img.shields.io/badge/GitHub-Repo-blue)](https://github.com/Vengefulcookie/ember-run)

## What This Project Does
You run through a dark forest. Lanterns float in the distance. Collect them to light your way. Each lantern brings you closer to safety — but the darkness pushes back. You have 3 lives. Run into danger and lose one. Collect all 10 lanterns and you win.

**What you need to know:**
- **Goal:** Collect 10 lanterns
- **Lives:** Start with 3, lose one when you hit danger
- **Atmosphere:** The screen gets brighter as you collect more lanterns
- **Sound design:** No sound by default, but the visuals carry the mood

## How I Built This
- **JavaScript** - Game logic
- **HTML5 Canvas** - Rendering and lighting effects
- **Game Loop** - For smooth movement
- **Collision Detection** - For lantern collection and danger avoidance

## What I Learned
This was my first game with atmosphere as a core mechanic. I wanted the player to feel the darkness closing in. I learned about dynamic lighting effects in Canvas — how to make the screen brighter as lanterns are collected. The collision system had to distinguish between good objects (lanterns) and bad ones (danger). It's amazing how a simple game can feel tense with the right visual design.
